Paradocs - Paraphrase で作られたドキュメントシステム

------------------------------------------------------------
これは一体何か？（開発の背景を含めて）
------------------------------------------------------------
　MkDocs や Sphinx など、世の中にはシングルソース・マルチユー
スに対応した優れたドキュメントシステムが既に多く存在していま
す。そのような状況なのに、なぜ更にまた新たなドキュメントシス
テムを作る必要があるのか？と思われる方もいらっしゃるとは思い
ます。

しかし、索引に対応したシングルソース・マルチユースなドキュメ
ントシステムは意外にも存在していません（単に私が見つけられな
かっただけなのかもしれませんが）。

　Paraphrase のワードリファレンス等、文量の多いドキュメント
の場合、索引は重要です。ちなみに現在のワードリファレンスはで
すが、25,000 行弱の HTML になっています。これは全て作者であ
る私が手作業で作成ならびに保守しています。新たなワードを追加
する場合等では、この HTML ファイルの保守・更新の作業が大きな
負担となっています。

そのため、ひとつの原稿ソースから索引付きの HTML ファイルなら
びに PDF ファイルを生成できるドキュメントシステムの存在は、
Paraphrase の開発においては必要不可欠な状況です。

　このような背景があり、Paradocs を開発するに至りました。言う
までもありませんが、Paradocs はシングルソース・マルチユースな
ドキュメントシステムであり、かつ、索引にも対応しています。基
本的に Paraphrase のワードリファレンスを生成するために開発し
ていますので、出力される文書ファイルは HTML と(TeX を経由した
）PDF ファイルとなります。

------------------------------------------------------------
使い方
------------------------------------------------------------
　Paradocs は Paraphrase にて記述されています。そのため Para-
docs は Paraphrase のサンプルプログラムとして提供もしています。
paradocs.p8e が Paradocs の本体プログラムです。
sanples/paradocs ディレクトリからの使用方法は

    ../../para paradocs.p8e document.pdconf

となります。pdconf は文書ファイルの生成について記述されたファ
イルです。例として同梱してあるのは v097.pdconf です。このファ
イルは ver. 0.97 のワードリファレンスを生成するためのものです。

なお、現時点では Paradocs によるワードリファレンスは未完成で
すが、途中までのところは contents ディレクトリ以下に格納して
ありますので、../../para paradocs.p8e v097.pdconf を実行すれ
ば、原稿が提供されている部分までの内容を記した
paraphrase_v097.html が samples/paradocs ディレクトリに生成
され、samples/paradocs/tmp に PDF 版のドキュメントを生成する
ための TeX ファイル（tmp.tex）が生成されます。あとはこの
tmp.tex を TeX でコンパイルすれば PDF 版のワードリファレンス
が完成します。

　contents ディレクトリには、まだ全てのワードを記した原稿が格
納されていませんが、ver. 0.98 では Paradocs で生成したワード
リファレンスが同梱できるよう頑張っていきたいと思います。
